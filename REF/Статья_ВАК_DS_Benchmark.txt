Аннотация.
В работе представлена система DS-Benchmark для воспроизводимого сравнения реализаций теории Демпстера—Шейфера в едином вычислительном конвейере. Предложена архитектура, включающая генерацию тестовых наборов, адаптерный слой, раннер сценариев и модуль профилирования. Используется унифицированный формат входных данных DASS на основе JSON, что обеспечивает сопоставимость экспериментов между различными библиотеками. Эксперименты организованы в виде четырехшагового процесса: вычисление исходных Belief/Plausibility, комбинирование по правилу Демпстера, дисконтирование с последующим комбинированием и комбинирование по правилу Ягера. Для каждого этапа собираются метрики времени, памяти, CPU и конфликта K. Представленный подход позволяет формировать единый воспроизводимый протокол сравнения и выявлять ограничения реализаций на конфликтных и масштабных наборах.

Ключевые слова: теория Демпстера—Шейфера, benchmark, DASS, профилирование, правило Демпстера, правило Ягера, конфликт K.

1. Введение
Теория Демпстера—Шейфера широко применяется в задачах принятия решений при неопределенности, интеграции данных из нескольких источников и интеллектуальном анализе информации. При этом на практике используются разные программные реализации, отличающиеся внутренними структурами данных, стратегиями комбинирования и вычислительной эффективностью. Отсутствие единого протокола тестирования затрудняет корректное сравнение таких реализаций.

Целью работы является разработка системы DS-Benchmark, обеспечивающей воспроизводимый и сопоставимый анализ производительности реализаций теории Демпстера—Шейфера.

2. Архитектура системы
Архитектура DS-Benchmark включает следующие компоненты:
- ядро операций теории Демпстера—Шейфера;
- адаптерный слой для унификации вызовов разных реализаций;
- генератор тестовых наборов;
- раннер четырехшагового тестового процесса;
- модуль профилирования и агрегации результатов.

Такое разделение обеспечивает переносимость сценариев между библиотеками и повторяемость вычислительных экспериментов.

3. Формат данных DASS
Для представления входных сценариев используется JSON-формат DASS (Dempster-Shafer Scenario Specification). Формат содержит:
- метаданные формата и версии;
- frame of discernment;
- список источников bba_sources;
- для каждого источника — отображение масс на подмножества гипотез.

Пример DASS (сокращенно):
{
  "metadata": {"format": "DASS", "version": "1.0"},
  "frame_of_discernment": ["A", "B", "C"],
  "bba_sources": [
    {"id": "source_1", "bba": {"{A}": 0.3, "{B}": 0.2, "{A,B}": 0.5}}
  ]
}

4. Методика экспериментов
Тестовый процесс включает четыре последовательных шага:
Шаг 1: расчет исходных Belief/Plausibility для каждого источника.
Шаг 2: комбинирование всех источников по правилу Демпстера.
Шаг 3: дисконтирование источников и повторное комбинирование по Демпстеру.
Шаг 4: комбинирование всех источников по правилу Ягера.

На каждом шаге измеряются:
- время выполнения (мс);
- пиковое потребление памяти (MB);
- средняя загрузка CPU (%);
- успешность выполнения;
- конфликт K (для операций комбинирования).

5. Результаты
В рамках базовой серии тестов (27 сценариев различной размерности) получены агрегированные оценки по этапам вычислений и подтверждена работоспособность конвейера для малых, средних и стрессовых наборов. Зафиксированы случаи полного конфликта (K=1.0), которые интерпретируются как корректное поведение в конфликтных постановках, а не как ошибка реализации.

Ключевой наблюдаемый эффект: с ростом размера frame of discernment нелинейно увеличиваются время выполнения и потребление памяти, что согласуется с комбинаторной природой операций над подмножествами гипотез.

6. Обсуждение
Предложенный подход позволяет:
1) стандартизировать сравнение различных библиотек;
2) выявлять узкие места отдельных этапов конвейера;
3) формировать доказуемо воспроизводимые экспериментальные отчеты.

Ограничения текущего этапа:
- перечень сравниваемых библиотек может быть расширен;
- результаты зависят от аппаратной платформы;
- при росте размерности возрастают вычислительные затраты.

7. Заключение
Разработана и апробирована система DS-Benchmark для воспроизводимого анализа производительности реализаций теории Демпстера—Шейфера. Практическая ценность состоит в наличии единого тестового конвейера, формата DASS и набора метрик, пригодных для академического сравнения и инженерной оптимизации.

Перспективы дальнейшей работы:
- расширение списка сравниваемых библиотек;
- углубленное профилирование (line-level, memory hotspots);
- автоматическая генерация публикационных отчетов.
