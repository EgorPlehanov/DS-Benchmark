# Results / Результаты

## RU
Каталог выходных артефактов запусков и отчетов.

### Основные разделы
- `profiling/` — результаты профилирования по библиотекам и запускам.
- `book_validation/` — результаты проверки на примерах из книги.

### Рекомендации
- Сохраняйте `run_id`-ориентированную структуру для воспроизводимости.
- Не редактируйте вручную сырье профилирования; делайте постобработку в `scripts/processing`.

---

## EN
Output artifact directory for benchmark runs and reports.

### Main sections
- `profiling/` — profiling outputs grouped by library and run.
- `book_validation/` — validation results for book examples.

### Recommendations
- Keep run-id-based structure for reproducibility.
- Avoid manual edits of raw profiling data; use `scripts/processing` for post-processing.
